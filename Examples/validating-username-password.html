Validating username and password

Lets say you username field which must contain a valid username. eg - test_123 or test123

and a password field which must contain a strong password eg.test@123 or Test@123

The password rule accepts parameters for number of upper and lower case and special characters and numbers.

If no parameter is provided it will default to 1 upper , 1 lower , 1 special character and 1 number
